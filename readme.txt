РУССКИЙ ===========================================================================
Выключатель назойливого сообщения в Яндекс.Браузере (возможно это приложение будет работать и для Chrome)  "Отключение расширений в режиме разработчика".

Приложение подойдёт для русскоязычной (а сейчас и для англоязычной) версии Яндекс.Браузера/Chrome  и возможно таких же хромоподобных, в которых появляется это сообщение.
После запуска приложение ищет такое сообщение и закрывает его за вас. Можно добавить в автозагрузку и сообщение  никогда не будет доставать.
Есть много вариантов решения этой проблемы, но я пошёл таким путём.

v. 1.1 Добавил поиск  сообщения "Disable developer mode extensions"

v. 1.2 Добавил поиск  сообщения "Disable developer mode<NBSP>extensions"

При просмотре кода в Notepad++ обнаружился символ «неразрывный пробел» в русской версии сообщения. На всякий случай добавил неразрывный пробел и к английской версии примерно в том же месте строки.

v. 1.3  Добавил поиск  сообщения "Отключение расширений в режиме разработчика"  без <NBSP> символа (неразрывного пробела)
Update 2023-12-02 в Windows 11 не работала 32-разрядная версия программы, я добавил 64-разрядную версию closevtel_x64.exe 

ENGLISH ===========================================================================

Google Translate

Disabling the annoying message in Yandex Browser (Maybe this app will work for Chrome too) "Отключение расширений в режиме разработчика".

The application is suitable for the Russian (and English now) version of Yandex.Browser/Chrome and possibly the same chromium-like ones, as part of the implementation of this message.
After launching the application, it looks for such a message and closes it for you. You can add it to autoload and the message will never reach.
There are many solutions to this problem, but I found this way.

v. 1.1 Added search messages "Disable developer mode extensions"

v. 1.2 Added search messages "Disable developer mode<NBSP>extensions"

When looking at the code in Notepad++, I found a "non-breaking space" character in the Russian version of the message. Just in case, I added a non-breaking space to the English version at about the same place in the line.

v. 1.3 Added search messages "Отключение расширений в режиме разработчика" without any <NBSP> symbol
Update 2023-12-02 the 32-bit version of the program did not work in Windows 11, I added the 64-bit version closevtel_x64.exe
